# TON Testnet Web Wallet

Веб-кошелёк для сети TON Testnet. Работает без собственного бэкенда - всё хранится локально в браузере.

## Функционал

- **Создание / импорт кошелька** — генерация новой seed-фразы (24 слова) или импорт существующей с валидацией.
- **Локальная защита** — seed-фраза шифруется паролем пользователя и хранится в `localStorage`.
- **Баланс и история транзакций** — входящие и исходящие TON-переводы с датой и адресом контрагента.
- **Поиск по истории** — фильтрация по адресу (входящему и исходящему) или хешу транзакции.
- **Receive** — отображение адреса, копирование в буфер, генерация QR-кода.
- **Send** — двухшаговый флоу: форма → экран подтверждения с адресом, суммой и **оценкой комиссии сети** (запрашивается с ноды через `estimateExternalMessageFee`).
- **Защита от Address Spoofing** — сверка вводимого адреса с историей; визуальная подсветка подозрительных совпадений по началу/концу адреса.

## Стек технологий

UI - React 19, TypeScript, Vite 
TON SDK - `@ton/ton`, `@ton/core`, `@ton/crypto` 
RPC - `@orbs-network/ton-access` — бесключевой доступ к публичным нодам Testnet 
Иконки - `lucide-react` 
Стили - Vanilla CSS с CSS-переменными, тёмная тема 
Тесты - `vitest`, `@testing-library/react` 

## Запуск локально

```bash
npm install
npm run dev
```

Приложение будет доступно по адресу `http://localhost:5173`.

## Тесты

```bash
npm run test
```

**43 теста** в 6 файлах:

`services/security.test.ts` - Валидация адресов, обнаружение address spoofing 
`services/wallet.test.ts` - Генерация мнемоник, валидация, шифрование/расшифровка, детерминированность keypair 
`components/CreateImportWallet.test.tsx` - Экраны CHOICE/LOGIN, навигация, предупреждения 
`components/ReceiveTON.test.tsx` - Рендер адреса, кнопка копирования, QR, Testnet-предупреждение 
`components/Dashboard.test.tsx` - Loading state, баланс, адрес, пустой список, поиск 
`components/SendTON.test.tsx` - Валидация формы, экран подтверждения, комиссия, успешная отправка
